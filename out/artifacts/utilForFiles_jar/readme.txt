INFO:

Имена файлов можно указывать как до опций, так и после. Расширение входных файлов ".txt". 
!!! Если в имени входных файлов существуют пробелы, требуется заменить их на символ "|", иначе считывание произойдёт некорректно.
Чтение файлов по умолчанию производится из корневой папки jar-файла, для изменения папки воспользуйтесь соответствующей опцией.

Доступные опции:
-o Задание директории для результатов. 
	!!! Если в пути существуют пробелы, требуется заменить их на символ "|", иначе считывание произойдёт некорректно.
	Пример 1: -o a\a|1\a11 (если внутри проекта). Файл будет сохранён в "...\[корневой каталог jar-файла]\a\a 1\a11"
	Пример 2: -o C:\b\b1 (для любого места на диске)
-p Задает префикс имен выходных файлов
	!!! Если в префиксе существуют пробелы, требуется заменить их на символ "|", иначе считывание произойдёт некорректно.
	Пример1: -p demo1_ Имя выходного файла: "demo1_floats.txt"
	Пример2: -p demo||3_ Имя выходного файла: "demo  3_floats.txt"
-a режим добавления в существующие файлы
-s краткая статистика (количество элементов в исходящие файлы)
-f полная статистика (для чисел: кол-во, мин, макс, сумма, среднее, для строк: кол-во, макс длина, мин длина.)
-ip расположение инпут файлов (нет в задании)
	!!! Если в пути существуют пробелы, требуется заменить их на символ "|", иначе считывание произойдёт некорректно.
	Пример 1: -o a\a|1\a11 (если внутри проекта). Файл будет взят из "...\[корневой каталог jar-файла]\a\a 1\a11"
	Пример 2: -o C:\b\b1 (для любого места на диске)
